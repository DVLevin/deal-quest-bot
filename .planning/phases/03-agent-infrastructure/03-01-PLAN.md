**Part A: Extend OpenRouterProvider in `bot/services/llm_router.py`**

Add two dataclasses at module level (after the existing `_extract_json` function, before the `LLMProvider` class):

```python
from dataclasses import dataclass

@dataclass
class TextResponse:
    content: str

@dataclass
class ToolCallResponse:
    tool_call_id: str
    tool_name: str
    tool_args: dict[str, Any]  # already JSON-parsed from the arguments string
    raw_message: dict[str, Any]  # full assistant message dict for appending to history
```

Add `complete_with_tools()` to `LLMProvider` ABC as an abstract method (not all providers need tool support — raise NotImplementedError in ClaudeProvider for now).

Add `complete_with_tools()` to `OpenRouterProvider`:

```python
@traced_span("llm:openrouter:tools")
async def complete_with_tools(
    self,
    system_prompt: str,
    messages: list[dict[str, Any]],  # full conversation history (user/assistant/tool messages)
    tools: list[dict[str, Any]],     # OpenAI-format tool definitions
    *,
    model: str | None = None,        # override model per-call (for per-agent model config)
    temperature: float = 0.7,
    max_tokens: int = 4096,
) -> TextResponse | ToolCallResponse:
```

Implementation:
- Build payload with `model` (use `model` param if provided, else `self.model`), `messages` prepended with system message, `tools`, `tool_choice: "auto"`, `max_tokens`, `temperature`.
- Use the same retry loop pattern as existing `complete()` (for loop over `MAX_RETRIES`, catch `httpx.HTTPStatusError` for 429/500/502/503, sleep `RETRY_DELAYS[attempt]`).
- Parse response: if `finish_reason == "tool_calls"` and `message.tool_calls` exists, return `ToolCallResponse` with the first tool call (parse `tc["function"]["arguments"]` with `json.loads()`). Otherwise return `TextResponse(content=message.get("content", ""))`.
- IMPORTANT: The `tools` array MUST be included in every request — OpenRouter requires it for the entire tool-use conversation.
- If max retries exhausted, return `TextResponse(content="I'm having trouble processing that right now. Please try again.")`.

For `ClaudeProvider`, add a stub: `async def complete_with_tools(self, ...) -> TextResponse | ToolCallResponse: raise NotImplementedError("Tool calling not yet supported for Claude provider")`.

**Part B: Create ToolUseAgent in `bot/agents/tool_use_agent.py`**

Create a new file `bot/agents/tool_use_agent.py` with:

```python
class ToolUseAgent:
    """Base class for agents that execute tool-use loops.

    Subclasses register tool handlers and invoke run_tool_loop().
    Phase 4+ specialist agents inherit from this.
    """
```

Constructor takes `AgentConfig` from `bot.agents.config`. Loads system prompt from `config.prompt_file` if the file exists (using `Path`, read_text). Initializes empty `_tool_handlers: dict[str, ToolHandler]` dict.

Properties:
- `name -> str`: returns `self.config.name`
- `config`: the `AgentConfig` instance

Methods:
- `register_tool(name: str, handler: ToolHandler) -> None`: Registers a callable async handler for a tool name. `ToolHandler = Callable[[dict[str, Any]], Awaitable[Any]]`
- `_build_tool_schemas() -> list[dict[str, Any]]`: Converts `self.config.tools` list of `ToolParam` into OpenAI tool format: `[{"type": "function", "function": {"name": t.name, "description": t.description, "parameters": t.parameters}}]`

The core method:
```python
@traced_span("agent:tool_use_loop")
async def run_tool_loop(
    self,
    messages: list[dict[str, Any]],  # conversation history as OpenAI messages
    llm_provider: OpenRouterProvider,  # the LLM provider with complete_with_tools
    *,
    system_prompt: str | None = None,  # override for the loaded prompt
) -> str:
```

Loop logic:
1. Use `system_prompt` param if provided, else `self._system_prompt`.
2. Build tool schemas from config.
3. Copy `messages` into local `history` list.
4. For each iteration up to `self.config.max_iterations`:
   a. Call `llm_provider.complete_with_tools(prompt, history, tools, model=self.config.model, temperature=self.config.temperature)`.
   b. If `TextResponse` → return `result.content` (loop complete).
   c. If `ToolCallResponse` → append `result.raw_message` to history, execute tool via `_execute_tool()`, append tool result message `{"role": "tool", "tool_call_id": ..., "content": str(tool_result)}` to history.
5. If max_iterations exhausted → log warning and raise `RuntimeError(f"Agent {self.name} exceeded max_iterations={self.config.max_iterations}")`.

`_execute_tool(tool_call: ToolCallResponse) -> Any`: Look up handler in `_tool_handlers` dict. If not found, return error string. If found, call `await handler(tool_call.tool_args)` wrapped in try/except (return error string on failure).

Do NOT subclass `BaseAgent` — `ToolUseAgent` is a separate class hierarchy for the v2.0 agent system. The existing `BaseAgent` is for the v1.0 pipeline agents (StrategistAgent, TrainerAgent, MemoryAgent). The two systems coexist.

Wire `@traced_span("agent:tool_use_loop")` on `run_tool_loop` so it participates in the existing tracing infrastructure from Phase 1.
