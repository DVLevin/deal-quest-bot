```sql
-- Used by ConversationHistoryService for hybrid in-memory + DB durability

CREATE TABLE conversation_history (
    id          BIGSERIAL PRIMARY KEY,
    telegram_id BIGINT NOT NULL,
    role        TEXT NOT NULL,              -- user | assistant | tool
    content     TEXT NOT NULL DEFAULT '',
    tool_calls  JSONB,                      -- assistant tool_calls array (for role=assistant)
    tool_call_id TEXT,                      -- for role=tool messages
    timestamp   TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Primary query pattern: get last N turns for a user, ordered by time
CREATE INDEX idx_conv_history_user_ts ON conversation_history(telegram_id, timestamp DESC);
```

**Part C: Repository**

Add `ConversationHistoryRepo` to `bot/storage/repositories.py` (at the end, after `TraceRepo`). Import `ConversationTurnModel` from models.

```python
class ConversationHistoryRepo:
    """Repository for conversation history persistence."""

    def __init__(self, client: InsForgeClient) -> None:
        self.client = client
        self.table = "conversation_history"

    async def get_recent(self, telegram_id: int, limit: int = 20) -> list[ConversationTurnModel]:
        """Get the most recent N turns for a user, ordered oldest-first."""
        rows = await self.client.query(
            self.table,
            filters={"telegram_id": telegram_id},
            order="timestamp.desc",
            limit=limit,
        )
        if rows and isinstance(rows, list):
            # Reverse to get chronological order (oldest first)
            return [ConversationTurnModel(**r) for r in reversed(rows)]
        return []

    async def save_turns(self, telegram_id: int, turns: list[ConversationTurnModel]) -> None:
        """Replace all stored turns for a user with the current window.

        Strategy: delete existing rows, insert current window.
        This is simpler and safer than diffing — the window is small (20 turns max).
        """
        try:
            await self.client.delete(self.table, {"telegram_id": telegram_id})
            for turn in turns:
                data = turn.model_dump(exclude_none=True, exclude={"id", "created_at"})
                data["telegram_id"] = telegram_id
                await self.client.create(self.table, data)
        except Exception as e:
            logger.error("Failed to save conversation history for user %s: %s", telegram_id, e)
```

The save strategy is delete-then-insert for the entire window. The window is small (max 20 turns), so this is efficient and avoids complex diffing logic. Background flush means this runs every ~30 seconds, not on every message.

**Part A: ConversationHistoryService**

Create `bot/services/conversation_history.py`:

The service manages per-user sliding window conversation history with hybrid persistence (in-memory deque for speed + async background flush to InsForge for durability).

```python
class ConversationTurn(BaseModel):
    """One message in conversation history."""
    role: str  # "user" | "assistant" | "tool"
    content: str = ""
    timestamp: str  # ISO 8601 UTC
    tool_calls: list[dict[str, Any]] | None = None
    tool_call_id: str | None = None
```

Note: `ConversationTurn` is the in-memory model used by the service. `ConversationTurnModel` in `storage/models.py` is the DB model. The service converts between them.

```python
class ConversationHistoryService:
    def __init__(
        self,
        history_repo: ConversationHistoryRepo,
        *,
        window_size: int = 20,
        flush_interval: float = 30.0,
        session_timeout_hours: float = 4.0,
    ) -> None:
```

State:
- `_cache: dict[int, deque[ConversationTurn]]` — user_id to bounded deque
- `_dirty: set[int]` — users with unflushed changes
- `_flush_task: asyncio.Task | None` — background flush loop
- `_loaded_users: set[int]` — tracks which users have been loaded from DB (for lazy loading)

Methods:

- `async start() -> None`: Creates background `_flush_task` running `_flush_loop()`.
- `async stop() -> None`: Cancels flush task, calls `_flush_all()` to persist any remaining dirty data.
- `async ensure_loaded(user_id: int) -> None`: Lazy-loads from DB on first access. If `user_id` not in `_loaded_users`, calls `self._repo.get_recent(user_id, limit=self._window)`, converts `ConversationTurnModel` list to `ConversationTurn` deque, stores in `_cache`, adds to `_loaded_users`. If already loaded, returns immediately.
- `get_messages(user_id: int) -> list[dict[str, Any]]`: Returns conversation history as OpenAI-format message list. Each turn becomes `{"role": turn.role, "content": turn.content}` plus optional `"tool_calls"` and `"tool_call_id"` fields. Does NOT call ensure_loaded (caller must call it first) — this is synchronous for performance.
- `append_turn(user_id: int, turn: ConversationTurn) -> None`: Appends to deque (auto-evicts oldest if at maxlen). Marks user as dirty. Creates deque with maxlen if not present.
- `clear_session(user_id: int) -> None`: Clears cache for user, marks dirty (will delete from DB on next flush).
- `detect_new_session(user_id: int) -> bool`: Returns True if last turn timestamp is more than `session_timeout_hours` ago. Returns False if no turns exist.
- `get_last_assistant_message(user_id: int) -> str | None`: Returns content of the last assistant turn (for session bridge summary). Returns None if no assistant turns found.

Background flush:
- `async _flush_loop() -> None`: `while True: await asyncio.sleep(flush_interval); await _flush_all()`. Wrap in try/except asyncio.CancelledError to exit cleanly.
- `async _flush_all() -> None`: For each user_id in `_dirty` (copy and clear set first), convert `ConversationTurn` deque to `ConversationTurnModel` list, call `self._repo.save_turns(user_id, turns)`. Wrap each user's flush in try/except to not fail all users if one fails.

Conversion helper `_turn_to_model(turn: ConversationTurn, telegram_id: int) -> ConversationTurnModel` and `_model_to_turn(model: ConversationTurnModel) -> ConversationTurn`.

Import `collections.deque`, `asyncio`, `datetime`, `logging`. Type `ToolHandler` not needed here.
